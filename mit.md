# MIT PD 模式原理与实现

## 控制律

```
τ = Kp × (p_des - p_cur) + Kd × (v_des - v_cur) + t_ff
```

| 参数 | 含义 | 主站每周期下发 |
|------|------|---------------|
| `p_des` | 期望位置 [rad] | ✅ |
| `v_des` | 期望速度 [rad/s] | ✅ |
| `Kp` | 位置刚度 [A/rad] | ✅ |
| `Kd` | 速度阻尼 [A·s/rad] | ✅ |
| `t_ff` | 前馈扭矩 [A] | ✅ |

## 与传统三环的区别

**传统三环**（位置→速度→电流）：
- 参数固定（Flash 里的 PID）
- 主站只发目标位置/速度
- 电机端做完整闭环
- 刚度/阻尼由 PID 参数决定，运行中不可变

**MIT PD**：
- 没有速度环、没有位置环 PID
- 主站每个控制周期（1~10ms）实时下发 5 个参数
- 电机端只做一步乘加 → 直接输出 Iq 给电流环
- 刚度/阻尼每拍可变 — 这是关键优势

## 实际效果

### 场景 1：高刚度定位（类似位置模式）
```
Kp=50, Kd=2, v_des=0, t_ff=0, p_des=目标位置
```
等效一个 PD 位置控制器，但没有积分项（不会 windup）。

### 场景 2：零刚度拖动示教
```
Kp=0, Kd=0.5, v_des=0, t_ff=0
```
电机只有微弱阻尼，人手可以自由拖动关节。

### 场景 3：轨迹跟踪 + 重力补偿
```
Kp=30, Kd=1, p_des=轨迹点, v_des=轨迹速度, t_ff=重力补偿扭矩
```
主站规划轨迹，前馈补偿已知力，PD 只修正误差。

### 场景 4：碰撞柔顺
```
正常: Kp=50, Kd=2
碰撞检测后: Kp=5, Kd=0.5  (瞬间变软)
```
主站检测到力突变，立刻降低刚度 → 关节变柔顺 → 安全。

## 为什么比传统方案好

1. **阻抗可变**：传统 PID 参数固定，MIT 每拍可调。机器人走路时腿着地要硬、摆动要软，MIT 一帧切换。
2. **无积分 windup**：纯 PD + 前馈，不会因为堵转积分饱和后弹射。
3. **通信带宽低**：5 个 float（或定点压缩到 12 字节）就够，不需要高频位置指令流。
4. **主站算力集中**：复杂的轨迹规划、动力学补偿、碰撞检测都在主站做，电机端极简。

## 本工程实现

### CAN 协议（万里扬 V1.7 0x500 帧）

12 字节/槽: `POS[23:0] + VEL[15:0] + T[15:0] + Kp[15:0] + Kd[15:0] + CANID`

解析位置: `can_wly.c handle_mit_cmd`

### 控制运算

位置: `foc_speed_loop.c` MIT_PD_MODE 分支（5kHz 速度环周期执行）

```c
float iq_out = controller->mit_kp * pos_err
             + controller->mit_kd * vel_err
             + controller->mit_t_ff;
```

### 工程改进（相对 motor_h7 原版）

| 改进 | 原因 |
|------|------|
| 速度低通滤波 `vel_filt += 0.05*(vel_raw-vel_filt)` | 减速箱速度反馈噪声大 |
| 位置死区 ±0.003 rad（≈0.17° 输出端）| 消除减速箱间隙引起的极限环振荡 |
| 电流限幅 `FlashData.MaxCurrent` | 防止大 Kp + 大偏差时电流冲击 |

### 单位

| 变量 | 单位 |
|------|------|
| `mit_p_des` / `pos_cur` | rad（输出端）|
| `mit_v_des` / `vel_cur` | rad/s（输出端）|
| `mit_kp` | A/rad |
| `mit_kd` | A·s/rad |
| `mit_t_ff` | A（q 轴电流）|
| 输出 `I_q_ref` | Q10（1024 = 1A）|

### 量程（CAN 协议默认）

```c
pos: ±7.0 rad (≈±401° 输出端)
spd: ±20.0 rad/s (≈±191 rpm 输出端)
tq:  ±500 N·m
Kp:  0~500
Kd:  0~20
```

## 典型应用

- 四足/双足机器人关节（MIT Cheetah、宇树 A1/Go1）
- 协作机械臂（需要力控/柔顺）
- 外骨骼（需要实时变刚度）
- 万里扬协议 0x500 帧，主站 100~500Hz 下发，电机端 5kHz 执行
