## NVIC 优先级分组

- **PriorityGroup**: `NVIC_PRIORITYGROUP_4`
- **说明**: 全部使用抢占优先级（PreemptPriority 0-15），子优先级（SubPriority）固定为 0
- **配置位置**: `stm32h7xx_hal.c:147` - `HAL_NVIC_SetPriorityGrouping(NVIC_PRIORITYGROUP_4)`

| `USART1_IRQn` | **1** | 0 | USART1 全局中断 (IDLE/ERROR) | 异步 | 调试命令 IDLE 中断 + 错误恢复 | `usart.c:204` |
| `FDCAN1_IT0_IRQn` | **1** | 0 | FDCAN1 中断线 0 | 异步 | 万里扬 FDCAN 协议收发 | `fdcan.c:117` |
| `TIM7_IRQn` | 15 | 0 | TIM7 更新事件 | 1 kHz | **HAL Timebase**（替代 SysTick） | `stm32h7xx_hal_timebase_tim.c:51` |

**注意事项**:
- 优先级数字**越小越高**（0 = 最高优先级，15 = 最低优先级）
- 同优先级中断**不会互相抢占**，按触发顺序排队执行
- `TIM7_IRQn` 的优先级由 `TickPriority` 参数决定（通常为 15），在 `HAL_InitTick()` 中设置

## DMA 流分配表

| DMA 流 | 外设 | Request | 方向 | DMA 优先级 | 模式 | 用途 | 代码位置 |
|--------|------|---------|------|-----------|------|------|---------|
| `DMA1_Stream0` | USART2_TX | `DMA_REQUEST_USART2_TX` | M2P | VERY_HIGH | NORMAL | DPT RS485 发送 | `usart.c:268-278` |
| `DMA1_Stream1` | USART2_RX | `DMA_REQUEST_USART2_RX` | P2M | VERY_HIGH | NORMAL | DPT RS485 接收 | `usart.c:250-260` |
| `DMA1_Stream2` | ADC2 | `DMA_REQUEST_ADC2` | P2M | LOW | NORMAL | ADC2 同步采样（无中断，从属 ADC1） | `adc.c:344-354` |
| `DMA1_Stream3` | USART1_TX | `DMA_REQUEST_USART1_TX` | M2P | LOW | NORMAL | printf 调试输出 | `usart.c:168-178` |
| `DMA1_Stream4` | USART1_RX | `DMA_REQUEST_USART1_RX` | P2M | LOW | NORMAL | 调试命令接收 | `usart.c:186-196` |
| `DMA2_Stream6` | ADC1 | `DMA_REQUEST_ADC1` | P2M | HIGH | CIRCULAR | ADC1 规则通道（VDC/温度） | `adc.c:290-300` |

**DMA 优先级说明**:
- `VERY_HIGH` > `HIGH` > `MEDIUM` > `LOW`
- DMA 优先级决定多个 DMA 流同时请求时的仲裁顺序
- NVIC 中断优先级决定 DMA 完成中断的响应优先级

## 当前配置的关键特征

### 1. FOC 关键路径（优先级 0）
```
TIM1 TRGO (10kHz) 
  → ADC1/2 注入采样 
  → ADC_IRQn (优先级 0) 
  → HAL_ADCEx_InjectedConvCpltCallback
  → Encoder_data_Calculate + Encoder_out_data_Calculate
  → FocOpenTest / MC_Loop_Schedule
```

### 2. DPT 编码器异步读取（优先级 0）
```
TIM1 CC4 (10kHz)
  → TIM1_CC_IRQn (优先级 0)
  → DPT_TriggerDualAngleWithStatusRead_Async()
  → USART2 TX DMA (DMA1_Stream0)
  → USART2 RX DMA (DMA1_Stream1)
  → HAL_UART_RxCpltCallback
  → 更新 g_latest_angles
```

### 3. 同优先级竞争问题
- `ADC_IRQn` 和 `TIM1_CC_IRQn` 都是优先级 0，**不会互相抢占**
- `DMA1_Stream1_IRQn`（DPT RX 完成）也是优先级 0
- 结果：`g_latest_angles` 更新与 FOC 读取**节拍不同步** → 负载端速度 0/2x 跳变

## CubeMX 配置建议（推荐方案）

### 方案 A：最小改动（保持当前架构）
保持 ADC 和 TIM1_CC 同优先级，通过**软件滤波**解决速度跳变：
- 给负载端速度 `dtheta_mech_out` 加滑动均值滤波（参照电机端 `Speed_Filter`）
- 优先级配置不变

### 方案 B：优先级重规划（推荐）
```
优先级 0:  ADC_IRQn                     (FOC 主回调 - 最高优先级，不可打断)
优先级 1:  TIM1_CC_IRQn                 (CC4 预触发 DPT，可被 FOC 抢占)
优先级 1:  DMA1_Stream0_IRQn            (USART2 TX DMA)
优先级 1:  DMA1_Stream1_IRQn            (USART2 RX DMA)
优先级 1:  USART2_IRQn                  (DPT IDLE/Err)
优先级 2:  TIM1_UP_IRQn                 (PWM 更新，可选)
优先级 2:  FDCAN1_IT0_IRQn              (CAN 通信)
优先级 3:  DMA1_Stream3_IRQn            (USART1 TX DMA)
优先级 3:  DMA1_Stream4_IRQn            (USART1 RX DMA)
优先级 3:  USART1_IRQn                  (调试串口)
优先级 3:  DMA2_Stream6_IRQn            (ADC 规则通道 1kHz)
优先级 15: TIM7_IRQn                    (HAL Timebase)
```

**优势**:
- ADC 回调 100 µs 内独占 CPU，不被任何中断打断
- DPT 链可以被 FOC 抢占，但不会被调试或 CAN 干扰
- 调试串口和慢速 ADC 不会影响实时性

## CubeMX 配置步骤

### 1. NVIC 配置
在 CubeMX 中打开 `.ioc` 文件：

**System Core → NVIC → Configuration**

| NVIC Interrupt | Enabled | Preemption Priority | Sub Priority |
|---------------|---------|---------------------|--------------|
| TIM1 capture compare interrupt | ✓ | 1 | 0 |
| TIM1 update interrupt | ✓ | 2 | 0 |
| ADC global interrupt | ✓ | 0 | 0 |
| USART1 global interrupt | ✓ | 3 | 0 |
| USART2 global interrupt | ✓ | 1 | 0 |
| FDCAN1 interrupt line 0 | ✓ | 2 | 0 |
| DMA1 stream0 global interrupt | ✓ | 1 | 0 |
| DMA1 stream1 global interrupt | ✓ | 1 | 0 |
| DMA1 stream3 global interrupt | ✓ | 3 | 0 |
| DMA1 stream4 global interrupt | ✓ | 3 | 0 |
| DMA2 stream6 global interrupt | ✓ | 3 | 0 |
| TIM7 global interrupt | ✓ | 15 | 0 |

### 2. DMA 配置
**Timers → TIM6 → DMA Settings**
- 确认 ADC1 规则通道使用 `DMA2 Stream 6`

**Connectivity → USART1 → DMA Settings**
- TX: `DMA1 Stream 3`, Priority: Low
- RX: `DMA1 Stream 4`, Priority: Low

**Connectivity → USART2 → DMA Settings**
- TX: `DMA1 Stream 0`, Priority: Very High
- RX: `DMA1 Stream 1`, Priority: Very High

**Analog → ADC1 → DMA Settings**
- 规则通道: `DMA2 Stream 6`, Priority: High, Mode: Circular

### 3. Timebase Source
**System Core → SYS → Timebase Source**
- 选择 `TIM7`（不要用 SysTick）
