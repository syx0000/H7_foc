## NVIC 优先级分组

- **PriorityGroup**: `NVIC_PRIORITYGROUP_4`
- **说明**: 全部使用抢占优先级（PreemptPriority 0-15），子优先级（SubPriority）固定为 0
- **配置位置**: `stm32h7xx_hal.c:147` - `HAL_NVIC_SetPriorityGrouping(NVIC_PRIORITYGROUP_4)`

## 当前中断配置表

| IRQ 名称 | 抢占优先级 | 子优先级 | 触发源 | 频率 | 用途说明 | 代码位置 |
|---------|-----------|---------|--------|------|---------|---------|
| `ADC_IRQn` | **0** | 0 | ADC1 注入转换完成 (JEOC) | 10 kHz | **FOC 主循环**：电流采样 + 编码器计算 + SVPWM | `adc.c:308, 362` |
| `TIM1_CC_IRQn` | **2** | 0 | TIM1 CH4 比较匹配 | 10 kHz | CC4 预触发 DPT 编码器异步读取 | `tim.c:248` |
| `USART2_IRQn` | **2** | 0 | USART2 全局中断 (IDLE/ERROR) | 异步 | DPT RS485 帧结束检测 / 错误恢复 | `usart.c:286` |
| `DMA1_Stream0_IRQn` | **2** | 0 | USART2 TX DMA 完成 | 异步 | DPT RS485 发送完成 → 切换到接收 | `dma.c:48` |
| `DMA1_Stream1_IRQn` | **2** | 0 | USART2 RX DMA 完成 | 异步 | DPT 应答接收完成 → 解析到 `g_latest_angles` | `dma.c:51` |
| `DMA2_Stream6_IRQn` | **4** | 0 | ADC1 规则通道 DMA 完成 | 1 kHz | VDC / 温度采样 DMA 循环完成 | `dma.c:60` |
| `FDCAN1_IT0_IRQn` | **6** | 0 | FDCAN1 中断线 0 | 异步 | 万里扬 FDCAN 协议收发 | `fdcan.c:117` |
| `USART1_IRQn` | **8** | 0 | USART1 全局中断 (IDLE/ERROR) | 异步 | 调试命令 IDLE 中断 + 错误恢复 | `usart.c:204` |
| `DMA1_Stream3_IRQn` | **8** | 0 | USART1 TX DMA 完成 | 异步 | printf DMA 发送完成 | `dma.c:54` |
| `DMA1_Stream4_IRQn` | **8** | 0 | USART1 RX DMA 完成 | 异步 | 调试命令接收（IDLE 组帧） | `dma.c:57` |
| `TIM7_IRQn` | 15 | 0 | TIM7 更新事件 | 1 kHz | **HAL Timebase**（替代 SysTick） | `stm32h7xx_hal_timebase_tim.c:51` |

**注意事项**:
- 优先级数字**越小越高**（0 = 最高优先级，15 = 最低优先级）
- 同优先级中断**不会互相抢占**，按触发顺序排队执行
- `TIM7_IRQn` 的优先级由 `TickPriority` 参数决定（通常为 15），在 `HAL_InitTick()` 中设置
- **当前配置已优化**: ADC_IRQn (优先级 0) 独占最高优先级，不会被任何中断打断
- DPT 编码器链 (TIM1_CC + USART2 + DMA) 为优先级 2，可被 FOC 抢占但不影响实时性

## DMA 流分配表

| DMA 流 | 外设 | Request | 方向 | DMA 优先级 | 模式 | 用途 | 代码位置 |
|--------|------|---------|------|-----------|------|------|---------|
| `DMA1_Stream0` | USART2_TX | `DMA_REQUEST_USART2_TX` | M2P | VERY_HIGH | NORMAL | DPT RS485 发送 | `usart.c:268-278` |
| `DMA1_Stream1` | USART2_RX | `DMA_REQUEST_USART2_RX` | P2M | VERY_HIGH | NORMAL | DPT RS485 接收 | `usart.c:250-260` |
| `DMA1_Stream2` | ADC2 | `DMA_REQUEST_ADC2` | P2M | LOW | NORMAL | ADC2 同步采样（无中断，从属 ADC1） | `adc.c:344-354` |
| `DMA1_Stream3` | USART1_TX | `DMA_REQUEST_USART1_TX` | M2P | LOW | NORMAL | printf 调试输出 | `usart.c:168-178` |
| `DMA1_Stream4` | USART1_RX | `DMA_REQUEST_USART1_RX` | P2M | LOW | NORMAL | 调试命令接收 | `usart.c:186-196` |
| `DMA2_Stream6` | ADC1 | `DMA_REQUEST_ADC1` | P2M | HIGH | CIRCULAR | ADC1 规则通道（VDC/温度） | `adc.c:290-300` |

**DMA 优先级说明**:
- `VERY_HIGH` > `HIGH` > `MEDIUM` > `LOW`
- DMA 优先级决定多个 DMA 流同时请求时的仲裁顺序
- NVIC 中断优先级决定 DMA 完成中断的响应优先级

## 当前配置的关键特征

### 1. FOC 关键路径（优先级 0，最高优先级）
```
TIM1 TRGO (10kHz) 
  → ADC1/2 注入采样 
  → ADC_IRQn (优先级 0) 
  → HAL_ADCEx_InjectedConvCpltCallback
  → Encoder_data_Calculate + Encoder_out_data_Calculate
  → FocOpenTest / MC_Loop_Schedule
```
**特点**: ADC ISR 独占最高优先级，不会被任何中断打断，保证 FOC 实时性

### 2. DPT 编码器异步读取（优先级 2）
```
TIM1 CC4 (10kHz)
  → TIM1_CC_IRQn (优先级 2)
  → DPT_TriggerDualAngleWithStatusRead_Async()
  → USART2 TX DMA (DMA1_Stream0, 优先级 2)
  → USART2 RX DMA (DMA1_Stream1, 优先级 2)
  → HAL_UART_RxCpltCallback
  → 更新 g_latest_angles
```
**特点**: 可被 ADC ISR 抢占，但不影响编码器数据更新（异步 DMA 传输）

### 3. 优先级分层设计
- **优先级 0**: FOC 主循环（ADC ISR），独占最高优先级
- **优先级 2**: 实时外设（DPT 编码器链），可被 FOC 抢占
- **优先级 4**: 慢速采样（VDC/温度 ADC DMA）
- **优先级 6**: 通信协议（FDCAN）
- **优先级 8**: 调试接口（USART1）
- **优先级 15**: 系统时基（TIM7 HAL Timebase）

## 历史优化记录

### 2026-05-13: 优先级重规划（已实施）
**问题**: 早期版本中 ADC_IRQn、TIM1_CC_IRQn、USART2 DMA 均为优先级 0，导致同优先级竞争，`g_latest_angles` 更新与 FOC 读取节拍不同步，负载端速度出现 0/2x 跳变。

**解决方案**: 
- ADC_IRQn 独占优先级 0（最高）
- DPT 编码器链降至优先级 2（TIM1_CC + USART2 + DMA）
- 调试接口降至优先级 8（USART1 + DMA）
- 其他外设按实时性分层（FDCAN=6, ADC规则=4）

**效果**: FOC 主循环不被任何中断打断，实时性得到保证；DPT 编码器异步读取可被 FOC 抢占但不影响数据更新。

## CubeMX 配置建议（当前已优化）

### 当前配置（推荐保持）
```
优先级 0:  ADC_IRQn                     (FOC 主回调 - 最高优先级，不可打断)
优先级 2:  TIM1_CC_IRQn                 (CC4 预触发 DPT，可被 FOC 抢占)
优先级 2:  DMA1_Stream0_IRQn            (USART2 TX DMA)
优先级 2:  DMA1_Stream1_IRQn            (USART2 RX DMA)
优先级 2:  USART2_IRQn                  (DPT IDLE/Err)
优先级 4:  DMA2_Stream6_IRQn            (ADC 规则通道 1kHz)
优先级 6:  FDCAN1_IT0_IRQn              (CAN 通信)
优先级 8:  DMA1_Stream3_IRQn            (USART1 TX DMA)
优先级 8:  DMA1_Stream4_IRQn            (USART1 RX DMA)
优先级 8:  USART1_IRQn                  (调试串口)
优先级 15: TIM7_IRQn                    (HAL Timebase)
```

**优势**:
- ADC 回调独占最高优先级，FOC 实时性得到保证
- DPT 链可以被 FOC 抢占，但异步 DMA 传输不受影响
- 调试串口和慢速 ADC 不会影响实时性
- 分层清晰，易于维护和扩展

## CubeMX 配置步骤

### 1. NVIC 配置
在 CubeMX 中打开 `.ioc` 文件：

**System Core → NVIC → Configuration**

| NVIC Interrupt | Enabled | Preemption Priority | Sub Priority |
|---------------|---------|---------------------|--------------|
| ADC global interrupt | ✓ | 0 | 0 |
| TIM1 capture compare interrupt | ✓ | 2 | 0 |
| USART2 global interrupt | ✓ | 2 | 0 |
| DMA1 stream0 global interrupt | ✓ | 2 | 0 |
| DMA1 stream1 global interrupt | ✓ | 2 | 0 |
| DMA2 stream6 global interrupt | ✓ | 4 | 0 |
| FDCAN1 interrupt line 0 | ✓ | 6 | 0 |
| USART1 global interrupt | ✓ | 8 | 0 |
| DMA1 stream3 global interrupt | ✓ | 8 | 0 |
| DMA1 stream4 global interrupt | ✓ | 8 | 0 |
| TIM7 global interrupt | ✓ | 15 | 0 |

### 2. DMA 配置
**Timers → TIM6 → DMA Settings**
- 确认 ADC1 规则通道使用 `DMA2 Stream 6`

**Connectivity → USART1 → DMA Settings**
- TX: `DMA1 Stream 3`, Priority: Low
- RX: `DMA1 Stream 4`, Priority: Low

**Connectivity → USART2 → DMA Settings**
- TX: `DMA1 Stream 0`, Priority: Very High
- RX: `DMA1 Stream 1`, Priority: Very High

**Analog → ADC1 → DMA Settings**
- 规则通道: `DMA2 Stream 6`, Priority: High, Mode: Circular

### 3. Timebase Source
**System Core → SYS → Timebase Source**
- 选择 `TIM7`（不要用 SysTick）
