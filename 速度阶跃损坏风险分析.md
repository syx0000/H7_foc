# 速度环从0直接给最大速度的硬件损坏风险分析

## 问题描述

电机速度环从静止（0 rpm）直接给定最大速度（100 rpm 输出端，对应电机端 2500 rpm），可能导致硬件损坏。

## 损坏机理分析

### 1. 电流冲击（最危险）

#### 1.1 速度环输出饱和
- **速度环 PID 输出** = 速度环的输出是电流环的给定值 `I_q_ref`
- **阶跃响应**：速度误差从 0 瞬间跳到 100 rpm × 25 × 1024 = 2,560,000 LSB
- **PI 输出饱和**：
  ```
  速度环 Kp = 1500, Ki = 10, Div = 65000
  比例项 = 2560000 × 1500 / 65000 = 59,077 (已超限幅)
  积分项快速累积，输出直接打到限幅 INC_PID_SPEED_LIMIT = 25A (Q10)
  ```
- **结果**：电流环瞬间收到 25A 的电流指令（额定电流的 3 倍以上）

#### 1.2 相电流尖峰
根据 CLAUDE.md 电流公式：
```
I = raw × 3.3V / 65535 / 10 / 0.0025Ω
16位ADC + 10倍运放 + 2.5mΩ 采样电阻
```

**25A 电流对应的功率**：
- 相电压 ≈ 48V / √3 = 27.7V（限幅 `INC_PID_CURRENT_LIMIT = 27.8A`）
- 瞬时功率 = 27.7V × 25A × 3相 = **2.08 kW**（远超 120W 额定）

**MOSFET 应力**：
- DRV8353RH 驱动能力：IDRIVEP ≈ 300mA / IDRIVEN ≈ 600mA
- 栅极充放电时间 ≈ 20ns（按 Qg=10nC 估算）
- **di/dt = 25A / 20ns = 1.25 × 10⁹ A/s**（极端值）
- 寄生电感 L_stray ≈ 10nH → 感应电压 = L × di/dt = **12.5V 尖峰**
- 可能击穿 MOSFET 栅极（Vgs_max 通常 ±20V）

#### 1.3 母线电容冲击
- 电机电感储能：E = 0.5 × L × I² = 0.5 × 0.113mH × (25A)² = **35.3 mJ**
- 母线电容 ≈ 100µF（CLAUDE.md 提到"测试架母线电容 ~100µF"）
- 电容电压波动：ΔV = E / C = 35.3mJ / 100µF = **353V**（理论值，实际会被限流）
- **风险**：母线电容 ESR 发热、电解液蒸发、寿命骤降

#### 1.4 采样电阻过载
- 采样电阻 R_sense = 0.0025Ω（2.5mΩ）
- 瞬时功率 = I² × R = (25A)² × 0.0025Ω = **1.56W**
- 如果持续 100ms（速度环响应时间 1/45.7Hz ≈ 22ms），累积能量 = 1.56W × 0.1s = **156 mJ**
- 0603 贴片电阻典型热容 ≈ 10 mJ/°C → 温升 ≈ **15.6°C**（单次阶跃）
- 多次阶跃会导致采样电阻漂移、烧毁

---

### 2. 机械冲击

#### 2.1 减速箱齿轮打齿
- **减速比 25:1**，电机端加速度 = 输出端 × 25
- 如果没有斜坡滤波，速度环直接给 100 rpm 输出端：
  - 电机端目标速度 = 100 × 25 = **2500 rpm**
  - 电机端加速度 ≈ 2500 rpm / 0.022s（速度环响应时间）= **113,636 rpm/s**
  - 折合角加速度 = 113636 × 2π / 60 = **11,900 rad/s²**

- **扭矩冲击**：
  - 惯量 J = 1.22 × 10⁻⁴ kg·m²（电机轴）
  - 冲击扭矩 = J × α = 1.22e-4 × 11900 = **1.45 N·m**（电机端）
  - 输出端扭矩 = 1.45 × 25 = **36.25 N·m**（远超额定扭矩 38 N·m）

- **齿轮啮合面应力**：
  - 齿轮模数 m ≈ 0.5mm（假设）
  - 接触应力 σ ∝ √(T / m³) → 冲击扭矩下应力可能超过材料屈服强度
  - **结果**：齿面点蚀、打齿、断齿

#### 2.2 编码器联轴器打滑
- DPT 双磁编码器通过联轴器连接
- 联轴器摩擦扭矩 ≈ 0.5 N·m（典型值）
- 冲击扭矩 1.45 N·m > 摩擦扭矩 → **联轴器打滑，编码器读数失真**
- 后果：位置环失控、电角度偏置漂移

#### 2.3 轴承寿命
- 轴承额定动载荷 C ≈ 500 N（小型电机）
- 冲击载荷 F = T / r = 1.45 N·m / 0.01m = **145 N**（径向分量）
- 冲击次数 N = 1 次 → 寿命损失 ΔL = (C / F)³ = (500/145)³ = **42 次等效循环**
- 频繁阶跃会导致轴承提前失效

---

### 3. 电气应力

#### 3.1 MOSFET 开关损耗
- 开关频率 f_sw = 20kHz（TIM1 PWM）
- 单次开关损耗 E_sw = 0.5 × V_ds × I_ds × (t_r + t_f)
  - V_ds = 48V, I_ds = 25A, t_r + t_f ≈ 40ns（DRV8353 典型值）
  - E_sw = 0.5 × 48 × 25 × 40e-9 = **24 µJ**
- 开关损耗功率 P_sw = E_sw × f_sw = 24µJ × 20kHz = **0.48W**（单管）
- 6 个 MOSFET 总损耗 = 0.48W × 6 = **2.88W**
- 加上导通损耗 P_cond = I²_rms × R_ds(on) ≈ (25A)² × 0.01Ω = **6.25W**
- **总损耗 ≈ 9W**（远超散热能力，MOSFET 结温可能超过 150°C）

#### 3.2 死区时间续流
- 死区时间 = 100ns（DRV8353RH 内部固化）
- 死区期间电流通过体二极管续流
- 二极管反向恢复电流 I_rr ≈ 2A（典型值）
- 反向恢复电荷 Q_rr = I_rr × t_rr ≈ 2A × 50ns = **100 nC**
- 反向恢复尖峰电压 = Q_rr / C_oss ≈ 100nC / 200pF = **500V**（可能击穿 MOSFET）

#### 3.3 母线电压振铃
- 母线寄生电感 L_bus ≈ 100nH（PCB 走线 + 电容引线）
- 母线电容 C_bus = 100µF
- LC 振荡频率 f_res = 1 / (2π√(LC)) = 1 / (2π√(100nH × 100µF)) = **50.3 kHz**
- 电流阶跃 25A → 振铃幅值 ΔV = I × √(L/C) = 25A × √(100nH / 100µF) = **0.79V**
- 叠加在 48V 母线上 → 峰值 48.79V（接近 OVP 阈值 60V）

### 4. 控制系统问题

#### 4.1 速度环积分饱和
- 速度误差 e = 2,560,000 LSB
- 积分项累积速率 = Ki × e / Div = 10 × 2560000 / 65000 = **394 LSB/拍**（5kHz 速度环）
- 积分限幅 = 20000 LSB（`func_pid.c` 硬编码）
- 饱和时间 = 20000 / 394 = **51 拍 ≈ 10ms**
- **后果**：速度超调、制动困难（积分项需要反向放电）

#### 4.2 电流环饱和
- 电流环限幅 `INC_PID_CURRENT_LIMIT = 27.8A`
- 速度环输出 25A → 电流环工作在限幅区
- **线性控制失效**：电流环 PI 输出被 clamp，无法精确跟踪
- 电角度快速变化 → Clarke/Park 变换误差增大

#### 4.3 位置环超调
- 如果位置环在外环，速度阶跃会导致：
  - 位置超调量 = ∫(v_error × dt) ≈ 100 rpm × 0.022s / 60 × 360° = **1.32°**（输出端）
  - 梯形规划的对齐判据 `labs(pos_err) < 50 LSB`（≈ 0.044°）
  - 超调后需要多次振荡才能收敛

### ✅ 已实现的保护

#### 1. 速度环斜坡滤波（`USE_SPEED_LOOP_SMOOTH = 1`）
```c
// foc_speed_loop.c:27-33
#if USE_SPEED_LOOP_SMOOTH
  controller->velocity_ref_filterd = SpeedLoopSmoothRun(
    controller->velocity_ref, 
    &controller->SpeedSmooth
```

**参数**：
- `MIN_ACC_TIME = 200 ms`（最小加减速时间）
- `MaxVelAccEveryPrd = DEFAULT_MAX_SPEED / MIN_ACC_TIME`
  - = (100 rpm × 25 × 1024) / 200ms
  - = 2,560,000 / 200 = **12,800 LSB/ms**
  - 折合加速度 = 12,800 / 1024 / 25 × 1000 = **500 rpm/s**（输出端）

**实际效果**：
- 从 0 加速到 100 rpm 需要 100 / 500 = **0.2 秒**

## 实测风险等级

### 高风险场景（可能立即损坏）

1. **冷启动全速阶跃**
   - 电机静止 → 100 rpm 输出端（2500 rpm 电机端）
   - 电流冲击 25A × 3 相 = 75A 峰值
   - **损坏概率**：MOSFET 击穿 > 50%，采样电阻烧毁 > 30%

2. **反向全速切换**
   - +100 rpm → -100 rpm（200 rpm 速度差）
   - 制动扭矩 = 加速扭矩 × 2 = **2.9 N·m**（电机端）
   - **损坏概率**：齿轮打齿 > 70%，编码器打滑 > 40%

3. **母线欠压时阶跃**
   - 母线 < 40V 时，电流环需要更高占空比
   - PWM 饱和 → 电流失控
   - **损坏概率**：过流 > 60%

### 中风险场景（累积损坏）

1. **频繁阶跃（> 10 次/分钟）**
   - 轴承疲劳、齿轮磨损
   - MOSFET 热循环应力
   - **寿命损失**：每次阶跃 ≈ 42 次等效循环

2. **高温环境（> 60°C）**
   - MOSFET 结温裕度降低
   - 电解电容 ESR 增大
   - **损坏概率**：过热 > 20%

### 低风险场景（斜坡滤波生效）

1. **斜坡滤波启用 + 加速度 ≤ 230 rpm/s**
   - 梯形规划默认 230 rpm/s（`POS_TRAPEZOID_DEFAULT_AMAX_RPS`）
   - 当前 500 rpm/s（`MIN_ACC_TIME = 200ms`）**偏快 2.17 倍**
   - 推荐调整到 400ms（对应 250 rpm/s，接近梯形规划）
   - 电流峰值 < 10A
   - **损坏概率**：< 1%

## MIN_ACC_TIME 参数推荐依据

### 计算公式

速度环斜坡滤波加速度与 `MIN_ACC_TIME` 的关系：

```
MaxVelAccEveryPrd = DEFAULT_MAX_SPEED / MIN_ACC_TIME
                  = (100 rpm × 25 × 1024) / MIN_ACC_TIME
                  = 2,560,000 / MIN_ACC_TIME  [LSB/ms]

实际加速度 = MaxVelAccEveryPrd / (1024 × 25) × 1000  [rpm/s]
           = 2,560,000 / MIN_ACC_TIME / 25,600 × 1000
           = 100,000 / MIN_ACC_TIME  [rpm/s]
```

**简化公式**：`加速度 [rpm/s] = 100,000 / MIN_ACC_TIME [ms]`

### 参数对照表

| MIN_ACC_TIME | 加速度 (rpm/s) | 0→100rpm 时间 | 与梯形规划对比 | 推荐度 |
|--------------|---------------|--------------|---------------|--------|
| 200 ms | 500 | 0.2 s | 2.17× 快 | ❌ 过快 |
| 300 ms | 333 | 0.3 s | 1.45× 快 | ⚠️ 可接受 |
| 400 ms | 250 | 0.4 s | 1.09× 快 | ✅ **推荐** |
| 434 ms | 230 | 0.434 s | 1.00× 对齐 | ✅ 最安全 |
| 500 ms | 200 | 0.5 s | 0.87× 慢 | ⚠️ 偏保守 |

**梯形规划参考**：`POS_TRAPEZOID_DEFAULT_AMAX_RPS = 230 rpm/s`（位置环）

### 推荐方案

#### 方案 A：对齐梯形规划（最安全）
```c
#define MIN_ACC_TIME 434  // 加速度 230 rpm/s，完全对齐位置环梯形规划
```
- **优点**：三环加速度一致，机械冲击最小
- **缺点**：响应稍慢（0.434s 加速时间）

#### 方案 B：平衡方案（推荐）
```c
#define MIN_ACC_TIME 400  // 加速度 250 rpm/s，略快于梯形规划
```
- **优点**：响应性好，安全性高，接近梯形规划
- **缺点**：比完全对齐稍快 9%

#### 方案 C：激进方案（不推荐）
```c
#define MIN_ACC_TIME 300  // 加速度 333 rpm/s，快于梯形规划 45%
```
- **优点**：响应快
- **缺点**：机械冲击大，与梯形规划不一致

#### 方案 D：当前配置（不推荐）
```c
#define MIN_ACC_TIME 200  // 加速度 500 rpm/s，快于梯形规划 2.17 倍
```
- **问题**：加速度过快，机械冲击大，与位置环不匹配

### 选择建议

1. **如果优先考虑安全性**：选择方案 A（434 ms）或方案 B（400 ms）
2. **如果需要平衡响应性和安全性**：选择方案 B（400 ms）✅
3. **如果已有完善的故障保护**：可考虑方案 C（300 ms）
4. **不推荐保持当前配置**（200 ms），除非：
   - 已实测验证无机械损坏
   - 已接入完整的过流/过压/过温保护
   - 减速箱和编码器联轴器经过加固

## 改进建议

### 短期（高优先级）

1. **降低速度环限幅**
   ```c
   // foc_controller.h:146
   #define INC_PID_SPEED_LIMIT (10 * 1024)  // 从 25A 降到 10A
   ```

2. **调整斜坡时间（对齐梯形规划）**
   ```c
   // foc_controller.h:38
   #define MIN_ACC_TIME 400  // 从 200ms 改为 400ms（加速度 250 rpm/s，接近梯形规划 230 rpm/s）
   ```
   
   **计算依据**：
   - 梯形规划加速度 = 230 rpm/s（`POS_TRAPEZOID_DEFAULT_AMAX_RPS`）
   - 对齐公式：`MIN_ACC_TIME = DEFAULT_MAX_SPEED / (目标加速度 × 1024 × 25 / 1000)`
   - = 2,560,000 / (230 × 1024 × 25 / 1000) ≈ **434 ms**
   - 推荐 **400 ms**（对应 250 rpm/s，略快于梯形规划，平衡响应性和安全性）
   
   **可选方案**：
   - 保守：`MIN_ACC_TIME = 500 ms`（200 rpm/s，更安全）
   - 激进：`MIN_ACC_TIME = 300 ms`（333 rpm/s，仍快于梯形规划）
   - 当前：`MIN_ACC_TIME = 200 ms`（500 rpm/s，**不推荐**，过快）

3. **接入故障保护**
   - 在 `main.c` while 循环加 1ms tick
   - 调用 `dcVoltageProFunc` / `busOverCurrentCheck` / `boradTempProFunc`
   - 故障时调用 `disablePWM()`

### 中期（中优先级）

4. **CAN 超时保护**
   ```c
   // can_wly.c 添加
   static uint16_t can_timeout_ms = 0;
   void CAN_TimeoutCheck(void) {  // 1ms 调用
     if (++can_timeout_ms > 100) {  // 100ms 超时
       disablePWM();
       controller_eyou.foc_run = 0;
     }
   }
   ```

5. **电流 RMS 计算**
   - 从 motor_h7 移植 `Calc_current_rms()`
   - 过载保护阈值 = 额定电流 × 1.5 × 持续 5s

6. **母线电压软限幅**
   ```c
   if (Vdc > 55V) {  // 提前 5V 触发
     INC_PID_SPEED_LIMIT = 5 * 1024;  // 降低电流限幅
   }
   ```

### 长期（低优先级）

7. **自适应斜坡**
   - 根据母线电压、温度、负载动态调整加速度
   - 参考 motor_h7 的 PP 模式

8. **VOFA+ 波形监控**
   - 实时监控 I_q / V_dc / 温度
   - 异常时自动保存波形

9. **硬件改进**
   - 增大母线电容（100µF → 470µF）
   - 添加刹车电阻（泄放制动能量）
   - DRV8353 nFAULT 引脚接入 GPIO
